Projects

FiscallyFit - Financial Literacy App

• Developed as a capstone project during my Computer Science undergraduate program

• Designed to improve financial literacy by helping users track, visualize, and understand personal spending habits.


Technologies Used

• Flutter: Built a responsive, cross-platform interface for both mobile and web using a single Dart codebase

• Dart: Used as the main programming language for the UI and business logic

• Plaid API: Integrated securely to fetch real-time financial data like account balances and transaction history from user-linked bank accounts

• Redis: Implemented caching to store the most recent API responses, reducing load times and minimizing unnecessary calls to Plaid

• Google Cloud Platform (GCP): Hosted backend services, enabling cloud scalability, managed infrastructure, and consistent uptime

• Cloud Functions/Instances: Used for API orchestration, auth token generation, background processing, and secure data handling


Key Features and Outcomes

• Achieved real-time financial data visualization through secure API handling

• Enabled offline performance optimization by caching frequent data requests

• Leveraged cloud infrastructure to build a scalable and production-ready app

• Applied secure authentication and session handling to protect sensitive user data

• Gained hands-on experience in fullstack development, DevOps, API management, and cloud deployment

BlueConnect - Bluetooth Multi-Link Manager

• Developed a Java/Kotlin-based mobile app that allowed one device to establish and manage simultaneous connections to multiple Bluetooth signals

• Designed to support use cases such as multi-device pairing for smart environments (e.g., fitness trackers, speakers, IoT sensors)

• Created as part of a collaborative engineering team, working alongside a dedicated UX Engineering group to refine layout, interactions, and accessibility


Technologies & Frameworks

• Java & Kotlin: Built the Android mobile app natively, leveraging Bluetooth APIs and concurrency tools

• Android Bluetooth Stack: Used BluetoothAdapter, BluetoothGatt, and BroadcastReceiver to manage connections, signal strength, and data flow

• Android Jetpack Components: Utilized LiveData, ViewModel, and Room for clean architecture and lifecycle-aware design

• Material Design Guidelines: Ensured a modern, polished interface through collaboration with the UX team


Team Collaboration & Process

• Worked in a cross-functional agile team: 3 software engineers + 2 UX designers + 1 product manager

• Participated in design sprints (Scrums), feedback loops, and UI testing sessions to iterate on wireframes and prototypes

• Practiced Git-based version control and code reviews to maintain team-wide coding standards and efficiency


Key Features & Outcomes

• Enabled multi-device Bluetooth pairing with a clean, user-friendly interface

• Real-time feedback on connection status, signal strength, and device metadata

• Focused on low energy optimization to ensure long-term Bluetooth session stability

• Delivered a fully functional prototype with strong UX principles and technical robustness

• Gained valuable experience in hardware-level interaction, native Android development, and design-engineering collaboration
